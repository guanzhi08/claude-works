📖遊戲規則

兩位玩家各自設定一組 4 位不重複數字（0–9），輪流猜對方的答案。第一個猜中 4A0B 者獲勝。

xAyB 判定系統

A = 數字正確且位置正確　　B = 數字存在但位置錯誤

答案：1234
1
3
2
7
猜：1 3 2 7
→
1A1B
1 位置對（1）
1 數字對位置錯（2）
勝利條件
猜出對方完整答案 → 結果為 4A0B，該玩家獲勝，遊戲立即結束。
回合規則
1
房主先攻
建立房間的玩家（Host）永遠先手，雙方嚴格交替回合。
2
每回合最多使用 1 張道具卡
道具可在猜數字前使用。使用後 G.usedTool = true，本回合不可再用。
3
猜完才換人
送出猜測後切換回合，對方收到結果才能繼續。
里程碑獎勵（雙人模式）

首次達成以下成就時，可從 2 張隨機道具卡中選 1 張：

里程碑條件旗標
⭐ 1A我的猜測首次出現至少 1AG.milestone[1]
⭐⭐ 2A我的猜測首次出現至少 2AG.milestone[2]
⭐⭐⭐ 3A我的猜測首次出現至少 3AG.milestone[3]

另外每 3 回合（G.myTurnCount % 3 === 0）也提供一次選卡機會。

⑤

1
Host 建立房間
產生 4 位房號 → 連接 WebSocket server，同時初始化 PeerJS（gn-{code}-h）。
2
Guest 加入
先嘗試 WebSocket（wss://omni-note.onrender.com/ws），5 秒未成功則降級 WebRTC。
3

4
雙方 Ready → 開始遊戲
各自送出 ready，checkBothReady() 確認雙方都好後呼叫 startGame()。
🃏道具卡系統
主動卡（每回合最多用 1 張）
🔍 審訊 ACTIVE
向對方詢問：「某個數字是否在答案中？」對方必須如實回答，獲得情報輔助下次猜測。
🛡 對方可用 Shield 抵擋 / 🪞 Reflect 反彈
💰 勒索 ACTIVE
向對方詢問：「第 N 位是什麼數字？」對方必須如實回答，直接得知一位數字。
🛡 對方可用 Shield 抵擋 / 🪞 Reflect 反彈
🌀 霧化 ACTIVE
對方下次猜測的結果只顯示「約 X 個數字正確」，不顯示 A / B 細節（G.fogOppNext）。
🛡 Shield 抵擋 / 🪞 Reflect 反彈效果（霧化自己下次結果）
🎯 聚焦 ACTIVE
我方下次猜測結果顯示每個數字的詳細狀態：A（位置對）、B（數字對）、X（不存在），逐位呈現（G.focusOppNext）。
🛡 Shield 抵擋 / 🪞 Reflect：聚焦效果施加到自己身上
🔐 保密 ACTIVE
此後對方看不到我方的猜測紀錄，對方的猜測 log 顯示「隱藏中」（G.secretActive）。
🛡 Shield 抵擋 / 🪞 Reflect：反轉效果，隱藏對方紀錄改為隱藏我方
被動卡（防禦，自動待機）
🛡 護盾 PASSIVE
對方使用任何道具卡時，可在 10 秒內選擇「護盾」完全抵消效果。一局限用 1 次（G.myShield）。
🪞 反射 PASSIVE
對方使用任何道具卡時，可在 10 秒內選擇「反射」將效果反彈給對方承受。一局限用 1 次（G.myReflect）。
反制時間窗
對方送出 play_tool 後，10 秒倒數視窗出現。玩家可選擇： 接受（效果生效）、護盾（消除）、反射（反彈）。 超時自動視為接受。

單人模式中電腦不會反制，可用卡限於：🔍 審訊、💰 勒索、🎯 聚焦（無霧化、保密、護盾、反射）。

WebSocket（主要） wss://omni-note.onrender.com/ws Host → ?room=gn-{code}&user=host Guest → ?room=gn-{code}&user=guest 成功 → transport = 'ws'，銷毀 PeerJS WebRTC / PeerJS（備援） Guest 5 秒內 WebSocket 失敗時啟動 Host PeerJS ID：gn-{code}-h Guest PeerJS ID：gn-{code}-g Guest 主動連線 → Host 接受 成功 → transport = 'webrtc'

特殊模式
測試模式
URL 加上 ?test=1 → 遊戲開始時立即獲得所有道具卡 + 護盾 + 反射，方便開發測試。
單人模式差異
  • 電腦答案隨機產生，不進行反制
  • 無聊天、無對方 log、無里程碑獎勵
  • 可用卡：審訊、勒索、聚焦（無霧化、保密）
  • 每 3 回合選卡，卡片立即可用
  • 顯示回合計數器代替「我的回合 / 對方回合」
